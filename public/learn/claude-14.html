Section 01

What is an API?

API sounds technical, but the concept is beautifully simple. If you understand ordering food at a restaurant, you already understand APIs. No coding knowledge needed.

🍽

The Restaurant Analogy

Imagine you're at a restaurant in Ludhiana:

You (the App) sit at a table and decide what you want.
The Menu (the API Documentation) tells you what's available and how to order.
The Waiter (the API) takes your order to the kitchen and brings back your food.
The Kitchen (Claude's Brain) prepares your order — you never see inside.
The Food (the Response) arrives at your table, ready to use.

An API is the waiter. It takes your request, sends it to Claude's brain, and brings back the answer. You never need to understand how the kitchen works. You just need to know how to order.

CONCEPT🔌

API = Application Programming Interface

It's a way for one piece of software to talk to another. When your weather app shows today's forecast, it's using a weather API. When you pay with UPI, your app uses a banking API. APIs are everywhere. The Claude API lets any app, website, or automation tool use Claude's intelligence.

WHY IT MATTERS🚀

Chat vs API

Claude Chat (claude.ai): You type, Claude responds. One person, one conversation, manual.
Claude API: Your app sends requests automatically. Can handle thousands of requests. No manual typing. This is how businesses scale AI.

Go to console.anthropic.com. Create an account. Navigate to API Keys. Click "Create Key." This key is your restaurant membership card — it identifies you and tracks your usage. Keep it secret. Never share it publicly. Anthropic gives you free credits to start experimenting.

Every app you love uses APIs. WhatsApp uses APIs to send messages. Zomato uses APIs to show restaurant menus. Google Maps uses APIs for directions. When you understand APIs, you understand how the digital world is actually built. And now you can build things too.

Understand This Deeply

The API is how Claude goes from being a chat tool (1 person at a time) to being a business engine (thousands of requests automatically). Every AI product — customer service bots, content generators, data analysers — uses the API. Understanding the API is understanding AI's real power.

Inside an API Request

Every API request is like a detailed food order. Let's look at the exact structure and understand every field. You don't need to write this — just understand it.

🧪 Demo 1: The API Request Structure (Annotated)

Ask Claude to explain the API request structure with colour-coded annotations. This is NOT coding — it's understanding:

📋 Click to copy: ਮੈਨੂੰ Claude API ਦੀ ਬਣਤਰ ਸਮਝਾਓ ਜੀ। Explain the Claude API JSON structure field by field to a non-technical person. model, max_tokens, temperature, system, messages. Restaurant analogy for each: model=chef, max_tokens=plate size, temperature=creativity, system=special instructions, messages=your order.

The 5 key fields explained:

model — Which Claude brain to use. Like choosing which chef cooks your food. "claude-sonnet-4-20250514" is the balanced model. Opus is the master chef (slower, more expensive). Haiku is the fast cook.

max_tokens — Maximum length of Claude's response. Like choosing your plate size. 1024 tokens is roughly 750 words. Set higher for longer responses, lower for quick answers.

temperature — How creative vs predictable Claude should be. Scale: 0.0 (exactly the same every time) to 1.0 (very creative, different each time). 0.7 is a good balance for most tasks.

system — Hidden instructions that control Claude's behaviour. Like telling the chef "this customer is vegetarian" before they see the order. The user never sees the system prompt.

messages — Your actual conversation. Each message has a "role" (user or assistant) and "content" (the text). This is the food order itself.

💡

JSON — Don't Be Scared

The API request is written in JSON (JavaScript Object Notation). It looks technical, but it's just a structured way to write information. Think of it like a form: field name on the left, value on the right, separated by a colon. Curly braces {} group related information. Square brackets [] make lists. You don't need to write JSON to understand the API.

Temperature & System Prompts

Temperature controls creativity. System prompts control behaviour. Together, they let you fine-tune Claude's output precisely for any use case.

🧪 Demo 2: Temperature Experiments

See how the same prompt gives different results at different temperatures. This demonstrates why temperature matters:

📋 Click to copy: Demonstrate temperature effect on Claude API output. Same prompt ("Write a tagline for Chai Wala, Kotkapura") at temperature 0.0, 0.5, 1.0. Show 3 runs each with example outputs. Explain when to use each, task suitability, and risks. Punjab-relevant examples.

Temperature guide: 0.0: Data extraction, classification, factual Q&A, spreadsheet parsing — consistency matters. 0.3-0.5: Business writing, emails, reports, marketing copy — professional with slight variation. 0.7-1.0: Brainstorming, poetry, creative writing, generating ideas — maximum creativity.

🌡

Temperature Cheat Sheet for Punjab Businesses

GST calculation? Temperature 0. You want the exact same answer every time.
Customer email reply? Temperature 0.3. Professional and consistent with slight personalisation.
Instagram caption? Temperature 0.7. Creative and engaging, different each time.
Brainstorming business ideas? Temperature 1.0. Wild, varied, surprising ideas to choose from.

They're the invisible instructions that make the same AI act completely differently for different purposes. When you build an AI product, the system prompt is where 80% of the value lives.
